一、安装Hugo

  1. 根据自己情况下载相应版本的 hugo ，并将解压后的文件路径添加到环境变量中。 下载地址
  2. 执行 hugo version 命令，当出现如下情况则说明安装成功。(以下所有指令均在Git Bash中运行)

二、创建博客

执行 hugo new site MyBlog 命令 （Moblog 为文件夹名，注意创建文件夹的路径）

三、设置主题

点击进入Hugo主题库，下载喜欢的主题，这里以 LoveIt 主题为例。

将 LoveIt 主题克隆到 themes 目录:

1
2

cd MyBlog/
git clone https://github.com/dillonzq/LoveIt.git themes/LoveIt

如果出现 fatal: unable to access ‘https://github.com/...‘ 错误，可以将 https 换成 git 试一下

进入该目录下的 themes 文件夹，发现已存在 LoveIt 文件夹，表示主题已经下载完成

修改 Moblog 目录下的config.toml文件，配置成自己想要的格式，如（可参考Moblog\themes\LoveIt\exampleSite路径下的``config.toml`文件）：

    post = ""
    name = "标签"

  [markup.highlight]
    # false 是必要的设置 (https://github.com/dillonzq/LoveIt/issues/158)
    noClasses = false

有关该主题更详细的设置请参考官方文档LoveIt (hugoloveit.com)

四、本地启动个人博客

调用指令 hugo server -t LoveIt --buildDrafts （或者hugo serve）

去查看 http://localhost:1313

五、创建第一篇文章，并浏览

1
2

hugo new posts/first_post.md
hugo serve

去查看 http://localhost:1313

六、部署博客到github中

  1. 建立仓库（注意：==箭头处名称应当相同且小写==）
  1. 输入以下代码
1
2
3
4
5
6
7

hugo --theme=LoveIt --baseUrl="https://xxxx.github.io/" --buildDrafts
cd public/
git init
git add .
git commit -m "第一次提交博客
git remote add origin https://github.com/xxx/xxx.github.io.git
git push -u origin master

  1. 输入用户名的密码

    结束部署。

七、（也可以）部署在gitee上

  1. 登录gitee，建立仓库

  2. 复制以下地址留着备用

  1. 打开config.toml，修改baseURL格式为：
1

baseURL = "https://用户名.gitee.io"
  1. 初始化Hugo
1

hugo --theme=LoveIt
  1. 进入public文件夹，并运行以下指令
1
2
3
4
5
6

cd public/
git init
git add .
git commit -m "第一次提交"
git remote add origin https://gitee.com/xxxxx/xxxxx.git
git push -u origin master
  1. 第一次运行，会弹出窗口，需要登录账号与密码 ==输错一次，需要重新开始==
  1. 刷新库，打开服务，强制使用HTTPS

  1. 完成部署
